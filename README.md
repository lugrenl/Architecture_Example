# Architecture Example (MVI Pattern)

Этот проект является демонстрационным шаблоном для обучения архитектуре мобильных приложений на Android с использованием паттерна **MVI (Model-View-Intent)** и принципов **Clean Architecture**.

## Особенности проекта
- **Jetpack Compose**: Современный UI-фреймворк.
- **State Management**: Использование `StateFlow` для управления состоянием экрана через единый `MainState`.
- **Clean Architecture**: Разделение на слои Presentation, Domain и Data.
- **Coroutines & Flow**: Асинхронная работа и реактивные потоки данных.
- **Simulated Backend**: Репозиторий имитирует сетевые задержки и случайные ошибки для демонстрации обработки состояний загрузки и ошибок.

## Структура проекта
- **data/**: Содержит модели данных (`Response`) и репозитории. Здесь реализована логика получения данных (в данном случае — из моков).
- **domain/**: Содержит бизнес-логику приложения. Интерактoры (Use Cases) обрабатывают данные для нужд презентационного слоя.
- **presentation/**: Слой UI.
    - `MainViewModel`: Управляет состоянием, обрабатывает "интенты" (команды) пользователя.
    - `MainState`: Единая модель состояния экрана (Loading, Success, Error).
    - `MainActivity`: Отрисовка UI на основе текущего состояния.

## Как это работает
1. UI подписывается на `state` в `MainViewModel`.
2. При запуске или нажатии кнопки "Обновить", вызывается метод `loadData()`.
3. ViewModel переключает состояние в `isLoading = true`.
4. Интерактoры запрашивают данные из репозитория.
5. При успешном получении данных состояние обновляется значениями из моделей.
6. В случае ошибки состояние обновляется текстом ошибки, который отображается пользователю с возможностью повтора.
