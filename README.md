# Architecture Example (MVI Pattern)

Этот проект является демонстрационным шаблоном для обучения архитектуре мобильных приложений на Android с использованием паттерна **MVI (Model-View-Intent)** и принципов **Clean Architecture**.

## Особенности проекта
- **Jetpack Compose UI**: Экран построен с использованием фундаментальных компонентов: `Column`, `Row`, `Box` и оптимизированного списка `LazyColumn`.
- **State Management**: Реактивное управление состоянием через `StateFlow` и единую модель `MainState`.
- **Clean Architecture**: Полное разделение ответственности:
    - **Domain-слой** содержит бизнес-логику и интерфейсы репозиториев (инверсия зависимостей).
    - **Data-слой** отвечает за реализацию интерфейсов и поставку данных (моки).
    - **Presentation-слой** отвечает за отображение и обработку действий пользователя.
- **Coroutines & Flow**: Асинхронное выполнение задач и работа с потоками данных.
- **Simulated Backend**: Имитация сетевых задержек и случайных ошибок для демонстрации работы состояний загрузки и обработки исключений.

## Структура проекта
- **data/**: Реализация источников данных.
    - `CardRepositoryImpl`: Конкретная реализация интерфейса репозитория.
    - `model/MainState`: Модель состояния экрана, используемая во всем приложении.
    - `CardMock`: Статичные данные для имитации работы API.
- **domain/**: Чистая бизнес-логика.
    - `repository/CardRepository`: **Интерфейс**, определяющий контракт для получения данных.
    - `GetCardInteractor` & `GetProductCountInteractor`: Интерактoры (Use Cases), выполняющие конкретные задачи.
- **presentation/**: Слой пользовательского интерфейса.
    - `MainViewModel`: **Stateful** компонент, управляющий состоянием экрана.
    - `MainActivity`: Декларативный UI на Compose, подписанный на изменения состояния.

## Как это работает
1. UI подписывается на `state` во ViewModel через `collectAsState()`.
2. При запуске или нажатии кнопки "Обновить состояние", вызывается метод `loadData()`.
3. ViewModel переводит состояние в `isLoading = true`.
4. Интерактoры запрашивают данные через интерфейс репозитория.
5. Репозиторий имитирует сетевой запрос, маппит данные в доменные модели и возвращает результат.
6. ViewModel обновляет `MainState` новыми данными (включая список особенностей товара для `LazyColumn`).
7. UI автоматически перерисовывается, отображая актуальное состояние.

## Стек технологий
